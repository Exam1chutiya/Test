SECTION D

Graph Traversal: BFS and DFS

(A) Definitions and Differences

Breadth-First Search (BFS)

BFS is a graph traversal algorithm that explores all neighbors of a node before going to the next level of neighbors. It is commonly implemented using a queue.

Depth-First Search (DFS)

DFS is a traversal algorithm that explores as far along each branch as possible before backtracking. It is commonly implemented using recursion or an explicit stack.

Key Differences

Criteria BFS DFS
(i) Approach Level-wise traversal Depth-wise traversal
(ii) Data Structure Used Queue Stack / Recursion
(iii) Time Complexity O(V + E) O(V + E)
(iv) Applications - Finding shortest path in unweighted graphs
- Level-order traversal in trees 		- Topological sorting
- Solving puzzles and mazes

(B) Graph Traversal Example

(i) Graph Representation

Vertices: A, B, C, D, E, F

Edges:

  • A - B
  • A - C
  • B - D
  • C - E
  • D - F
  • E - F

Graph Diagram (Text-based):

         A
        / \
       B   C
       |    \
       D     E
        \   /
          F

(ii) BFS Traversal (Starting from A)

Step 1: Start at A → Enqueue A → Visited: [A] → Queue: [A]
Step 2: Dequeue A → Visit B, C → Visited: [A, B, C] → Queue: [B, C]
Step 3: Dequeue B → Visit D → Visited: [A, B, C, D] → Queue: [C, D]
Step 4: Dequeue C → Visit E → Visited: [A, B, C, D, E] → Queue: [D, E]
Step 5: Dequeue D → Visit F → Visited: [A, B, C, D, E, F] → Queue: [E, F]
Step 6: Dequeue E → No unvisited neighbors → Queue: [F]
Step 7: Dequeue F → Done

BFS Order: A → B → C → D → E → F

(iii) DFS Traversal (Starting from A)

Step 1: Start at A → Visit A
Step 2: Go to B → Visit B
Step 3: Go to D → Visit D
Step 4: Go to F → Visit F
Step 5: Backtrack → D → B → A
Step 6: Go to C → Visit C
Step 7: Go to E → Visit E → Done

DFS Order: A → B → D → F → C → E

(iv) Comparison of Traversal Orders

BFS: A → B → C → D → E → F

DFS: A → B → D → F → C → E

When to Prefer BFS or DFS?

  • BFS: Preferred when shortest path or minimal number of steps is needed (e.g., pathfinding in unweighted graphs).
  • DFS: Preferred in scenarios requiring complete exploration (e.g., solving puzzles, detecting cycles, topological sorting).

Q2: Student Records Using Array of Structures

C Program with Sorting

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    int roll;
    char name[50];
    float marks;
} Student;

void input(Student *s, int n) {
    for (int i = 0; i < n; i++) {
        printf("Enter roll, name, marks for student %d:\n", i + 1);
        scanf("%d", &s[i].roll);
        getchar(); // clear buffer
        fgets(s[i].name, 50, stdin);
        s[i].name[strcspn(s[i].name, "\n")] = 0; // remove newline
        scanf("%f", &s[i].marks);
    }
}

void insertionSort(Student *s, int n) {
    for (int i = 1; i < n; i++) {
        Student key = s[i];
        int j = i - 1;
        while (j >= 0 && s[j].marks < key.marks) {
            s[j + 1] = s[j];
            j--;
        }
        s[j + 1] = key;
    }
}

void display(Student *s, int n) {
    printf("\nSorted Student List (Descending Marks):\n");
    for (int i = 0; i < n; i++) {
        printf("Roll: %d | Name: %s | Marks: %.2f\n", s[i].roll, s[i].name, s[i].marks);
    }
}

int main() {
    int n;
    printf("Enter number of students: ");
    scanf("%d", &n);

    Student *students = (Student *)malloc(n * sizeof(Student));
    if (!students) {
        printf("Memory allocation failed!\n");
        return 1;
    }

    input(students, n);
    insertionSort(students, n);
    display(students, n);

    free(students);
    return 0;
}

Explanation of Techniques Used

  • Structures: Used to group student fields like roll, name, and marks.
  • Dynamic Memory Allocation: malloc() allocates memory for n students at runtime.
  • Pointers: Array of structures accessed using pointers to students.
  • Structure Passing: Structures are passed to functions like input(), display(), and insertionSort() by reference (using pointers).

Q4: Why is Insertion Sort More Efficient than Bubble Sort on Small or Nearly Sorted Arrays?

  • Insertion Sort: Efficient for small/nearly sorted arrays because it shifts elements only when needed.
  • Bubble Sort: Repeatedly swaps adjacent elements regardless of overall sortedness, leading to more operations.
  • Time Complexity: Best-case for insertion sort is O(n), while bubble sort is always O(n²).

struct Node {
    int data;
    Node *left, *right;
};

void inorderIterative(Node* root) {
    Stack s;
    Node* current = root;

    while (current != NULL || !s.isEmpty()) {
        while (current != NULL) {
            s.push(current);
            current = current->left;
        }

        current = s.pop();
        printf("%d ", current->data);
        current = current->right;
    }
}
