Section A – Q A2

Phishing: Attackers trick users with fake emails/websites into giving away credentials.

Pharming: Attackers redirect users to fake websites by manipulating DNS, even with correct URL.

Section B – Q B4

Brute-force attack:

  • Online: Slow and detectable, tries passwords on live system.
  • Offline: Fast and dangerous; cracks stolen password hashes.

Replay attack: Reusing captured login packets.

Weak implementation attack: Exploiting outdated crypto, bad key management.

Section C – Q C1 (a)

Static Analysis: Examines malware without running it; fast but fails against obfuscation.

Dynamic Analysis: Executes malware in sandbox; reveals behavior but resource-heavy.

Impact: Static is quick but limited; dynamic is accurate but slower; hybrid is ideal.

Section C – Q C1 (b)

Major Challenges in Malware Detection:

  • Polymorphic & metamorphic malware
  • Fileless malware
  • Zero-day vulnerabilities
  • Sandbox evasion
  • Supply chain attacks
  • Encrypted C2 channels
  • Rapid mutation via malware kits
  • Insider threats
